深入原理：Spring 底层

一、Spring Bean 生命周期底层

1. AbstractApplicationContext.refresh() 流程

refresh() 是 Spring 启动核心方法：

1. prepareRefresh()
   - 初始化前置处理
   - 验证必要属性

2. obtainFreshBeanFactory()
   - 创建/获取 BeanFactory
   - 加载 BeanDefinition

3. prepareBeanFactory(beanFactory)
   - 配置 BeanFactory
   - 注册默认后处理器

4. invokeBeanFactoryPostProcessors(beanFactory)
   - 执行 BeanFactoryPostProcessor
   - 处理顺序：PriorityOrdered → Ordered → 无顺序

5. registerBeanPostProcessors(beanFactory)
   - 注册 BeanPostProcessor

6. initMessageSource()
   - 初始化消息源（国际化）

7. initApplicationEventMulticaster()
   - 初始化事件广播器

8. registerListeners()
   - 注册监听器

9. finishBeanFactoryInitialization(beanFactory)
   - 初始化非懒加载 Bean（核心）

10. finishRefresh()
    - 清理缓存
    - 发布 ContextRefreshedEvent

2. getBean() 流程底层

AbstractBeanFactory.getBean():

1. 转换 name
   - alias → beanName
   - FactoryBean 前缀处理

2. 检查缓存
   - singletonObjects：已创建的单例
   - earlySingletonObjects：早期单例
   - singletonFactories：单例工厂

3. 检查循环依赖
   - singletonsCurrentlyInCreation：正在创建的单例集合
   - 在集合中 = 循环依赖

4. 标记创建中
   - beforeSingletonCreation(beanName)

5. 创建 Bean
   - createBean(beanName, mbd, args)

6. 处理循环依赖
   - 三级缓存机制

7. 完成创建
   - afterSingletonCreation(beanName)

3. doCreateBean() 流程

doCreateBean():

1. createBeanInstance()
   - 实例化 Bean

   a. 工厂方法
      - factory-method 创建

   b. 构造器注入
      - resolveConstructorArguments()：解析构造参数
      - autowireConstructor()：自动选择构造器

   c. 无参构造
      - 反射或 CGLIB 创建实例

2. applyMergedBeanDefinitionPostProcessors()
   - 收集注入元数据（@Autowired、@Resource）

3. addSingletonFactory()
   - 解决循环依赖
   - 注册 ObjectFactory 到三级缓存

4. populateBean()
   - 属性注入
   - 处理 @Autowired、@Resource、@Value

5. initializeBean()
   - 初始化 Bean

6. registerDisposableBean()
   - 注册销毁逻辑

4. initializeBean() 流程

initializeBean():

1. invokeAwareMethods()
   - BeanNameAware.setBeanName()
   - BeanClassLoaderAware.setBeanClassLoader()
   - BeanFactoryAware.setBeanFactory()

2. applyBeanPostProcessorsBeforeInitialization()
   - ApplicationContextAwareProcessor：处理 ApplicationContextAware
   - InitDestroyAnnotationBeanPostProcessor：处理 @PostConstruct

3. invokeInitMethods()
   - InitializingBean.afterPropertiesSet()
   - XML 配置的 init-method

4. applyBeanPostProcessorsAfterInitialization()
   - AbstractAutoProxyCreator：创建 AOP 代理
   - AsyncAnnotationBeanPostProcessor：创建异步代理

5. 三级缓存实现

三级缓存：

一级：singletonObjects（ConcurrentHashMap）
- 完整的单例 Bean

二级：earlySingletonObjects（HashMap）
- 早期单例对象（属性未注入）

三级：singletonFactories（HashMap）
- 单例工厂（ObjectFactory）

流程：

1. A 创建
   - 标记 singletonsCurrentlyInCreation.add(A)
   - 实例化 A（属性未注入）
   - 注册 singletonFactories.put(A, () -> getEarlyBeanReference(A))

2. A 注入 B
   - getBean(B)
   - B 创建

3. B 创建
   - 实例化 B
   - 注册 singletonFactories.put(B, ...)

4. B 注入 A
   - getBean(A)
   - A 在 singletonsCurrentlyInCreation 中
   - 循环依赖！

5. 从缓存获取 A
   - singletonObjects：没有
   - earlySingletonObjects：没有
   - singletonFactories：有
   - 执行 ObjectFactory.getObject()
   - 返回早期 A（可能已代理）
   - 放入 earlySingletonObjects
   - 移除 singletonFactories

6. B 注入早期 A
   - B 完成
   - singletonObjects.put(B)

7. A 注入 B
   - B 已在 singletonObjects
   - A 完成
   - singletonObjects.put(A)
   - 移除 earlySingletonObjects(A)

二、AOP 底层原理

1. AbstractAutoProxyCreator

AbstractAutoProxyCreator：核心 AOP 后处理器

postProcessAfterInitialization():
1. 检查是否已处理
2. wrapIfNecessary(bean, beanName, cacheKey)

wrapIfNecessary():
1. getAdvicesAndAdvisorsForBean()
   - 查找适用的 Advisor
   - 返回拦截器数组

2. createProxy()
   - 创建代理对象

getAdvicesAndAdvisorsForBean():
1. findCandidateAdvisors()：找所有候选 Advisor
2. findAdvisorsThatCanApply()：筛选适用的 Advisor
3. sortAdvisors()：排序

2. JdkDynamicAopProxy

JdkDynamicAopProxy：JDK 动态代理实现

字段：
- AdvisedSupport advised：代理配置
  - targetSource：目标对象
  - advisors：Advisor 列表
  - interfaces：接口列表

invoke(Object proxy, Method method, Object[] args):
1. equals/hashCode/toString：特殊处理

2. getInterceptorsAndDynamicInterceptionAdvice()
   - 获取拦截器链

3. ReflectiveMethodInvocation.proceed()
   - 执行拦截器链

3. 拦截器链执行

ReflectiveMethodInvocation:

字段：
- proxy：代理对象
- target：目标对象
- method：方法
- args：参数
- interceptorsAndDynamicMethodMatchers：拦截器列表
- currentInterceptorIndex：当前索引

proceed():
if (currentInterceptorIndex == interceptors.size() - 1)
    return invokeJoinpoint();  // 执行目标方法

Object interceptor = interceptors.get(++currentInterceptorIndex);
return interceptor.invoke(this);

拦截器类型：
- MethodBeforeAdviceInterceptor：执行前置通知，然后 proceed()
- AfterReturningAdviceInterceptor：proceed()，然后执行返回通知
- ThrowsAdviceInterceptor：try { proceed() } catch，执行异常通知
- AspectJAfterAdvice：proceed()，然后执行后置通知

三、事务底层原理

1. TransactionInterceptor

TransactionInterceptor：事务拦截器

invoke(MethodInvocation invocation):
1. getTransactionAttribute()
   - 获取事务属性

2. determineTransactionManager()
   - 确定事务管理器

3. invokeWithinTransaction()
   - tm.getTransaction(txAttr)
   - invocation.proceedWithInvocation()
   - commitTransactionAfterReturning()

2. DataSourceTransactionManager.doBegin()

doBegin():

1. 获取 Connection
   Connection con = dataSource.getConnection()

2. 设置隔离级别
   con.setTransactionIsolation(isolationLevel)

3. 设置只读
   if (readOnly) con.setReadOnly(true)

4. 关闭自动提交
   con.setAutoCommit(false)

5. 创建 ConnectionHolder
   holder = new ConnectionHolder(con)

6. 绑定到线程
   TransactionSynchronizationManager.bindResource(dataSource, holder)
   - resources（ThreadLocal<Map<Object, Object>>）
   - dataSource → ConnectionHolder

3. 事务传播行为实现

PROPAGATION_REQUIRED（默认）：
- 已有事务：加入
- 无事务：创建新事务

PROPAGATION_REQUIRES_NEW：
- 总是创建新事务
- 挂起当前事务
- 保存 suspendedResources

PROPAGATION_NESTED：
- 创建嵌套事务（Savepoint）
- con.setSavepoint()

四、自动配置底层原理

1. @EnableAutoConfiguration

@EnableAutoConfiguration:
@Import(AutoConfigurationImportSelector.class)

AutoConfigurationImportSelector:
selectImports():
1. getCandidateConfigurations()
   - SpringFactoriesLoader.loadFactoryNames()
   - META-INF/spring.factories
   - key: EnableAutoConfiguration

2. removeDuplicates()
   - 去重

3. getExclusions()
   - 排除配置

4. filter()
   - OnClassCondition
   - OnBeanCondition
   - OnPropertyCondition
   - 过滤不满足条件的配置

2. Condition 实现

@ConditionalOnClass：
- OnClassCondition.matches()
- ClassUtils.isPresent(className, classLoader)
- 检查类是否存在

@ConditionalOnBean：
- OnBeanCondition.matches()
- BeanFactory.containsBeanDefinition(beanName)
- 检查 Bean 是否存在

@ConditionalOnProperty：
- OnPropertyCondition.matches()
- Environment.getProperty(key)
- 检查属性值

@ConditionalOnMissingBean：
- 检查 Bean 不存在
- 通常用于默认配置
